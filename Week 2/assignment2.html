Commonly Used Command on Git and Usage

1. Git add- It is used for after checking the status of the files, to add those files to the staging area

2. Git init- It is used to create an empty repository.

3. Git clone- It creates a local working copy of the source code from a remote repository

4.Git push- It pushes the committed file changes from the local repository to the remote repository and also create a named branch in the remote repository.

5. Git log- It is used to examine the detailed log of every commit in our repository.

6. Git rm- It is used to remove files from a repository.

7. Git diff- It Is often used alog with git status and git log commands to analyze the current state of our Git repository

8. Git tag- This allows developers to assign a permanent name and version number to a specific commit in the git repository.

9. Git stash- It allows developers to temporarily save changes to the working directory without committing them to the git repository

10. Git fetch- It is used to retrieve files from a Git repository

11. Git fsck- It is used to check the integrity of the git file system and it also helps in identifying corrupted objects.

12. Git rebase- It lets you move branches around, which helps you avoid unnecessary merge commits

13. Git branch- It lets you create isolated development environments within a single repository.

14. Git pull- It updates local respository to the newest commit.

15. Git checkout- It is used mostly for switching from one branch to another, it is also used for checking out files and commits.

16. Git commit- It takes the staged snapshot and commits it to the project history.

17. Git revert- It is used to undo our commit.

18. Git merge- It integrates your feature branch with all of its commits back to the dev master.

19. Git clean- It removes untracked files from the working directory.

20. Git commit--amend- It lets you amend the most recent commit.

21. Git reflog- It keeps track of updates to the tip of branches using a mechanism called reflog.

22. Git reset- It undoes changes to files in the working directory.

23. Git rebase-i- It is used to begin an interactive rebasing session.

24. Git config- It is a convenient wat to set configuration options for your Git installation.

25. Git show- This shows the metadata and content changes of the specified commit.

26. Git pull origin- It pulls changes from remote repository

27. Git push origin --delete- This deletes a remote branch.

28. Git notes- It adds or inspect object notes.

29. Git restore- It restores working tree files.

30. Git mv- It moves or rename a file, a directory, or a symlink.

31. Git switch- It is used to switch branches.

32. Git mergetool - It runs merge conflict resolution tools to resolve merge conflicts.

33. Git worktree - It manages multiple working trees.

34. Git submodules- It initialize, update or inspect submodules.

35. Git difftool - It shows changes using common diff tools.

36. Git range-diff - It compares two commit ranges.

37. Git shortlog - It summarize git log output.

38. Git describe - It gives an object a human readable name based on an available ref.

39. Git apply - It apply a patch to files and/or to the index.

40. Git help - It displays help information about Git.

41. Git bugreport - It collects information for user to file a bug report

42. Git cherry-pick - It apply the changes introduced by some existing commits.

43. Git blame - It shows what revision and author last modified each line of a file.

44. Git grep - It prints lines matching a pattern.

45. Git am - It applies a series of patches from a mailbox

46. Git format-patch - It prepares patches for e-mail submission.

47. Git send-email - It sends a collection of patches as emails.

48. Git request-pull - It generates a summary of pending changes.

49. Git svn - Bidirectional operation between a subversion repository and Git.

50. Git - It moves objects and refs by archive.
